Introduction

Fundamentals: Color, Typography, Spacing, Sizing and Imagery. Common Patterns: Headers, Hero Areas, Buttons, Image Galleries, Contact form and more. visit envato elements. Key takeaways:

  • A web designer is in charge of designing the front-end of a website or web application
  • A UX designer is in charge of creating the front-end of a website or web app from a user’s perspective. UX (user experience) includes things like information architecture, user psychology, usability testing content strategy, wireframes and interaction design.
  • A UI designer is in charge of the front-end but covers aspects that relate almost exlusively to the visual appearance of the project. UI stands for user interface so it all about things like color, typography, spacing, sizing, shape and so on. Both UX and UI design are 2 separate areas of specialisation within web design. it’s correct to say that you are a UX web designer or UI web designer or both.

Project Brief:

  • Always start a web project with a brief. In the brief be clear about what you’re making and who are you making it for.
  • Be as detailed as possible regarding the final content and include that in the brief. The project brief will serve as a basis for the wireframe.

Wireframes:

  • A wireframe is a low fidelity version of the final design.
  • A wireframe should display the final content as well as it’s structure.
  • A wireffame should be devoid of any intentional aesthetic decision like color or typography.

Picking the right design tool: For UI design avoid photoshop and illustrator and focus on using a more appropriate app. Sketch, Adobe XD and Figma are great options to consider, each having their pros and cons. My recommendation is Figma because it’s free, cross-platform and works in the browser.

Color

Color theory:

  • There are 3 primary colors: Red, Yellow and Blue. These represent the building blocks for all the other colors.
  • The secondary colors are obtained by mixing the primary ones. Red + Yellow = Orange, Red + Blue = Purple, Blue + Yellow = Green.
  • Tertiary colors are obtained by mixing primary and secondary colors.
  • A hue is the dominant family of a given color.
  • Tints, shades and tones are obtained by mixing a color with white, black and grey respectively.
  • The CMYK color model (cyan, magenta, yellow, black) is used for printing.
  • The RGB (red, green, blue) and HSB (hue, saturation, brightness) models are used in digital media.
  • The HEX (hexadecimal) color format is a more friendlier of representing RGB colors.

Readability One very important factor in font choise is readability so after you've chosen a couple of fonts that you think would work well with your project, think about how those fonts would look like in different sizes especially smaller sizes because if you're displaying all this text content of your website on a mobile device and the text is really small and people can't read it, then what do you do. Loading time is something that we often forget about and the thing is The more fonts or font weights we use in our design the slower a loading time of that website is going to be and the bigger the impact on the performance and yes i know that website performance is on the web development side of things so technically not our job but if you're sending a developer a design that requires them to load six or seven different fonts or font weights and the website performance has to suffer because of that, well that's on you because you should have thought about that, there is an overlap between two different specialisations web development and web design you going to be mindful about these things and always have the bigger picture in you mind. Technically you're the designer but you must also design the website in such a way that's not going to pose problems to the developers. Finally we're talking about Font attributes and roles let's say that you picked your font or fonts for your project. Next you need to decide on the font rules and their attributes and roll meaning what font will be used for what element for the headings what will you choose for paragraphs, what will it be and attribute meaning seze, height, line height and so on. In regards to roles it's very simple when you're using a single font family for the entire website because all the elements will be using fonts from that family. But when you're using two different font families or as they're also called typefaces, then it gets a bit trickier. Personally i like to use to fonts with the greatest readability for smaller elements in my UI like body text, buttons, menus, captions and that sort of things and i'll use the other typefaces on larger elements like headings so that takes care of roles. Key takeaways:

  • To properly define the typography of your project, there are a few steps you need to follow.
  • First: where you want to get your fonts from (I recommend Google Fonts or Adobe fonts) then decide which fonts would work best for the barnd and audience, base on the massage you want your website to convey. Fonts should also be picked using criteria like readability and loading time.

Spacing and Sizing

For beginner dong use grid in figma instead use ruler, and this is key tekeaways

  • Spacing and sizing are very important in UI design because they will dictate the amound of whitespace and also the visual hierarchy.
  • For spacing and sizing elements, use the 8pt system where every margin, padding, width etc. is a multiple of 8.

  • A lot of hero areas show sliders, but usually they display a strong message and a call-to-action

Buttons

Key takeaways:

  • in UI design, a button is a fundamental interactive element that serves as a middleman between the user and the product or service. ecamples: avocode, notion, adidas, timelyapp, youtube
  • Buttons are not used solely for navigation, they can perform various action like toggling an element's visibility, performing a task in the back-end etc.

Footer

Key takeaways:

  • A website footer is an area positioned at the bottom of every page on a website, right below the main content examples: adidas, grovemade, stripe, avocode, timelyapp
