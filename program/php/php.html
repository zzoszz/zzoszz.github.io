Perbedaan OOP (object oriented programming) dan PP (procedural programming)

PP

  • Instruksi yang dilakukan langkah demi langkah
  • Memecah program program menjadi bagian bagian kecil (disebut subroutine/function)
  • linear / top to bottom
  • fortran, algol, cobol, pascal, c, php, javascript

Kelebihan procedural programming

  1. to the point
  2. simplicity dan kemudahan implementasi (untuk compiler dan interpreter)
  3. mudah ditelusuri
  4. membutuhkan lebih sedikit memori (dibandingkan dengan oop)

OOP

  • Menyusun semua kode program dan struktur data ke dalam objek
  • objek adalah unit dasar dari program
  • objek menyimpan data dan prilaku
  • objek bisa saling berinteraksi
  • java, ruby, python, c++, js, php5

Kelebihan oop:

  1. representasi dunia nyata
  2. enkapsulasi dan abstraksi data
  3. reusability
  4. skalabilitas dan ekstensibilitas
  5. kemudahan pengelolaan dan kolaborasi
  6. digunakna oleh framework

basic oop

  • class & objek
  • property dan method
  • constructor
  • object type
  • inheritance
  • visibility dan access modifier
  • setter dan getter
  • static method

advanced oop

  • abstract dan interface
  • interceptor
  • object cloning
  • callbacks $ closures
  • namespaces dan autoloading

Istilah:

  • class: blueprint/template untuk membuat instance dari objek, mendefinisikan objek, menyimpan data dan prilaku yang disebut dengan property dan method
  • object: instance yang didefinisikan oleh class, banyak objek yang dapat dibuat menggunakan satu class, objek dibuat menggunakan keyword new
